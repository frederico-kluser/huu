- [2026-06-24][source:inference][task:greedy-max-mode][probation] Adding a scaler mode beyond auto/manual touches TWO non-obvious spots besides AutoScaler: (1) the per-tick target recompute in executeTaskPool is gated by `getMode() === 'auto'` — a new mode that needs a dynamic instanceCount must be added to that condition or it stays frozen at its seed; (2) the mode's enable method MUST call `portAllocator.setMaxAgents(AUTO_SCALE_MAX_INSTANCES)` like enableAutoScale(), else real concurrency is silently pinned at the manual port window (MAX_INSTANCES=20) however high targetConcurrency() returns. - [2026-06-24][source:inference][task:greedy-max-mode][probation] AutoScaleStatus.enabled is `this.enabled && mode === 'auto'` — it means "auto is DRIVING the target", NOT "scaler is on". - [2026-06-25][source:inference][task:card-action-counters][probation] To derive a per-agent accumulator from the agent event stream (e.g. per-action counts + lastAction for the kanban label), `handleAgentEvent` is the single funnel — bump it at the TOP, BEFORE the switch, via a map-mutate-WITHOUT-emit helper (`bumpAction`, mirrors `appendAgentLog`): read `this.agents.get(id)`, spread a new object, `this.agents.set`, and rely on `handleAgentEvent`'s trailing `this.emit()`.
