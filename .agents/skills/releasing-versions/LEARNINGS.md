- [2026-06-25][source:inference][task:npm-deploy-v2.0.0][probation] `npm publish` (`huu-pipe`) requires a 2FA OTP (account = "auth and writes"). NOT detectable beforehand: `npm whoami` succeeds, `npm access get status` only shows public/private, `npm profile get` doesn't reveal it — it surfaces only at publish time as `npm error code EOTP`. The agent can't generate it: have the user run `npm publish --otp=<6-digit>` via the `!` session-command prefix (fresh code, ~30s window; a placeholder/expired code re-errors EOTP). `npm publish` does NOT touch git, so a failed OTP leaves a clean tree to retry.
- [2026-06-25][source:user][task:npm-deploy-v2.0.0][probation] Versioning at 1.x+: a BREAKING change (removing the `huu --web` front-end) shipped as a MAJOR bump (1.4.0 → 2.0.0), NOT minor. The "0.x.x convention: breaking → minor" line in CLAUDE.md/CHANGELOG/this skill governed only the 0.x phase; past 1.0.0, breaking rides MAJOR (scripts/deploy.ts's own prompt already says "major — breaking changes em 1.0.0+"). User explicitly chose 2.0.0. The CHANGELOG header convention line is now stale for 1.x+.
- [2026-06-25][source:inference][task:npm-deploy-v2.0.0][probation] ALWAYS `rm -rf dist && npm run build` before `npm publish` after any source REMOVAL: `tsc` never prunes orphaned outputs and `prepack` only strips `*.test.*`/`*.spec.*`, so a stale `dist/` keeps compiled files whose source was deleted. Caught here: `dist/web/**` + `dist/cli-web.*` (the removed web mode) were about to ship inside 2.0.0. Gate the real publish behind `npm publish --dry-run` + `find dist -path '*web*'` (or equivalent) to confirm the tarball matches current source.
