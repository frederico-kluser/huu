- [2026-06-12][source:inference][task:tui-ux-redesign][probation] Full-screen pick panels with their OWN useInput (ListPick pattern) need the parent's useInput to early-return while a panel is open (same gate as pickingFiles/pickingModel) — otherwise both handlers fire per keystroke.
- [2026-06-24][source:inference][task:greedy-max-mode][probation] A new run-screen hotkey (RunDashboard) is just another branch in the existing ref-stable useInput handler reading stateRef.current — no new ref needed, deps stay `[orch]`. BUT when a key NUDGES a value the orchestrator recomputes every poll tick (concurrency under auto/greedy), the handler must first pin manual or the nudge is invisibly overwritten next tick: the +/- "pin manual" guard had to change from `autoScale.enabled` to `autoScale.mode !== 'manual'` so it also drops out of the new greedy/MAX mode (enabled is false in any non-auto mode, so the old guard no-opped). Header state chips are mutually exclusive on `mode` (MAX) vs `enabled` (AUTO) vs `!enabled && mode!=='greedy'` (GUARD); MAX uses theme.info (blue) — a perf knob, never theme.ai/magenta.
- [2026-06-25][source:inference][task:tui-options-and-welcome-polish][probation] A full-width footer hint bar is `<Box width="100%" justifyContent="space-between" flexWrap="wrap">` with one `<Box marginRight={1}>` per hint; per-key semantic colors (G=theme.success, ESC=theme.error bold for the two most important; rest theme.info) read far better than the old uniform `dimColor` line. Extracted as a shared `ActionBar`/`ActionHint` component (src/ui/components/ActionBar.tsx) reused by PipelineEditor + OptionsScreen — neither uses magenta.
- [2026-06-25][source:inference][task:tui-options-and-welcome-polish][probation] Auto-routing the TUI on a backend auth failure: throw a typed `AuthError {code:'AUTH', backendKind, specName}` (src/lib/auth-error.ts) from the orchestrator's pre-run reachability probe (pi) / Azure factory on 401/403; RunDashboard's `orch.start().catch` detects it via `isAuthError(err)` and calls a new `onAuthError(specName)` prop (ref-mirrored like onComplete/onAbort) instead of `setError`, which dispatches a `run.authError` FSM event → `{kind:'options', focusSpecName}`. Reliable for the pi pre-flight probe (rejects start()); Azure's per-agent factory throw surfaces as an agent error, not necessarily a start() rejection.
