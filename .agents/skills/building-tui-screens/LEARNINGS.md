- [2026-06-25][source:inference][task:card-action-counters][probation] RunKanban cardHeight() ↔ Card sync, refined: a row only costs a budget line if it's a NEW conditional `<Text>`; MERGING data into an existing always-rendered row (folded the "→ lastAction" marker into the always-present `log:` line) costs ZERO extra rows — only the separate counters label needed `if (card.actionsLabel) lines += 1`. Nested `<Text>` works inside a `<Text color="gray" dimColor wrap="truncate-end">`: the inner `<Text color={...}>` renders NON-dim (dimColor doesn't inherit to child Text nodes) while a trailing string sibling stays dim, and truncate-end trims from the END, so a SHORT colored leading token (`→ tool`) is truncation-safe — same idiom as SavedPipelinesManager's `<Text dimColor><Text bold>Y</Text> confirm</Text>`. Per-action telemetry colors map to the STATUS palette (error=red, done=green, stream/tool=cyan, else gray), NOT theme.ai — deterministic agent telemetry is not "an LLM did this" UI.
