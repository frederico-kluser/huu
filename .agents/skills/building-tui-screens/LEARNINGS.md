- [2026-06-12][source:inference][task:tui-ux-redesign][probation] Full-screen pick panels with their OWN useInput (ListPick pattern) need the parent's useInput to early-return while a panel is open (same gate as pickingFiles/pickingModel) — otherwise both handlers fire per keystroke.
- [2026-06-24][source:inference][task:greedy-max-mode][probation] A new run-screen hotkey (RunDashboard) is just another branch in the existing ref-stable useInput handler reading stateRef.current — no new ref needed, deps stay `[orch]`. BUT when a key NUDGES a value the orchestrator recomputes every poll tick (concurrency under auto/greedy), the handler must first pin manual or the nudge is invisibly overwritten next tick: the +/- "pin manual" guard had to change from `autoScale.enabled` to `autoScale.mode !== 'manual'` so it also drops out of the new greedy/MAX mode (enabled is false in any non-auto mode, so the old guard no-opped). Header state chips are mutually exclusive on `mode` (MAX) vs `enabled` (AUTO) vs `!enabled && mode!=='greedy'` (GUARD); MAX uses theme.info (blue) — a perf knob, never theme.ai/magenta.
- [2026-06-25][source:inference][task:web-ui][probation] The Ink TUI is no longer the only front-end: `src/web/` is a SECOND presentation layer (sibling to `ui/`, may import orchestrator/lib/models, never the reverse) and the web UI is now the DEFAULT — `huu` opens the browser, `huu --cli`/`HUU_CLI=1` keeps the TUI. cli.tsx forks AFTER the git + subcommand gates (so `auto`/`status`/`init-docker`/`--help` are unaffected): web → `startWebServer()` (mirrors `render(<App/>)`), else the Ink render path is unchanged. The browser surfaces the SAME `OrchestratorState` the kanban does — agents/`stageIntegrations`/`checkRuns` become cards, `agent.logs` feed the click-through drawer — over one throttled SSE stream (`orch.subscribe` → ~120ms coalesce, same spirit as headless-run's NDJSON throttle). The web client is plain HTML/CSS/JS under `src/web/client/`, served by a `node:http` server and copied to `dist/web/client` by an added build step (tsc ignores non-TS); resolved at runtime via `new URL('./client/', import.meta.url)` (works in dev-tsx AND dist). theme.ai=magenta is an Ink-only rule; the web theme uses Apple systemIndigo/Purple as accent, which honors the same "AI = purple" identity.
