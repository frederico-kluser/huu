- [2026-07-01][source:inference][task:dogfood-angular-react-multirun][probation] A CheckStep judge run with a WEAK model (openai/gpt-4o-mini) emitted no parseable verdict JSON, so huu logged "judge produced no parseable verdict; using default outcome" and took the `default: true` outcome. With the recommended-safe pattern `default: approved`, that means the judge SILENTLY RUBBER-STAMPS — it never actually gates. In a real Angular→React refactor this let genuine defects through (empty `dependencies` in package.json, a missing `<script src=main.jsx>` entry so the app never mounts, an invalid 7-digit hex color). Lessons: (a) give judge/CheckStep steps a capable model (`modelId` override on the step) — the verdict-JSON discipline is exactly where small models fail; (b) keep the `condition` mechanically checkable so even a weak judge can comply, and consider making the loop-back (`rework`) the more likely path when unsure; (c) NEVER trust a green judge as proof — inspect the diff yourself. The judge's forward-default is a safety valve against infinite loops, not evidence the work is correct.
