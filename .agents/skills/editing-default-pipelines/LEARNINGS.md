- [2026-06-28][source:user][task:test-suite-code-frozen][probation] The Test Suite default shipped a source-edit escape hatch ("if a REAL bug is exposed, fix `$file`" in step3 + step5) — a test-gen pipeline must NEVER edit the code under test (the dominant LLM test-gen failure mode is cheating to green by editing the SUT). Replace with characterization (pin ACTUAL behavior, stays green) + a fixed-shape `suspected-bug` FAQ finding + a strict xfail-family marker. Enforcing it needs TWO mechanisms: a CLEANUP step that ALWAYS runs once before the judge and mechanically restores drift — `git checkout $baseCommit -- <path>` for M/D AND `git rm -f <path>` for ADDED files (checkout errors on a path absent at base and leaves it) — plus a judge that diffs `$baseCommit..HEAD` fail-closed. Detection alone does NOT close it: never-rewind + the forward-default outcome let a smuggled committed edit persist past `maxRuns`. `registry.test`'s REPORT-ONLY assertion is AUDIT_NAMES-only so the Test Suite is exempt — pin its code-frozen contract in a colocated `huu-test-suite.test.ts` instead. The residual (all layers are LLM-executed) is honest; a deterministic orchestrator protected-paths gate is the real close, deferred because a generic "what is a test file" allowlist is per-repo and a wrong cut false-fails legit runs.
