- [2026-06-25][source:user][task:openrouter-key-401-shadow][promoted] "Valid key still 401s" is almost always the resolver picking a DIFFERENT key than the one the user set: env var (step 3) outranks the Options/config.json store (step 4), so a stale `OPENROUTER_API_KEY` (commonly exported from a shell profile or a sourced `~/.secrets`) shadows what the user saved. Diagnose with `checkOpenRouterReachable` vs a raw `curl https://openrouter.ai/api/v1/auth/key` ÔÇö same endpoint, so a 200 on curl + 401 in huu means key MISMATCH, not a bad key. `dev`/`start` run native (`HUU_NO_DOCKER=1`), so the resolver sees the host env directly.
- [2026-06-25][source:inference][task:openrouter-key-401-shadow][promoted] Added `resolveApiKeyWithSource(spec) -> {value, source, shadowsStored}` + `keyRemedyHint(spec, res)` in `api-key.ts` (resolveApiKey now delegates; value unchanged). The orchestrator 401 path and the docker-reexec host loop use them to name the ACTUAL source instead of the misleading "update it in Options" (a no-op when an env var shadows the saved key). `shadowsStored` = a higher tier won AND a different non-empty key sits in the store.

- [2026-06-25][source:user][task:invert-key-precedence][probation] SUPERSEDES the precedence half of the two `[task:openrouter-key-401-shadow]` entries above: the resolver was INVERTED so the explicitly SAVED store now OUTRANKS the env var. New order in `resolveApiKeyWithSource`: secret-mount Ôćĺ **stored** Ôćĺ env-file Ôćĺ env Ôćĺ none. The `shadowsStored` field was renamed to `storedOverridesEnv` (true only when `source==='stored'` AND a different non-empty ambient env value exists ÔÇö an ignored env var). `keyRemedyHint`'s foot-gun message moved from `case 'env'` to `case 'stored'` (env now only reached as the fallback when nothing is saved). Rationale: explicit user choice beats ambient shell leakage; CI is unaffected (no Options save Ôćĺ store empty Ôćĺ env still wins) and Docker is safe (`~/.config` isn't mounted, so in-container the secret-mount carries the host-resolved key). The web already preferred the browser key (`run-manager.ts` `params.apiKey || resolveApiKey`), so only the terminal resolver changed. Consumers updated: `docker-reexec` host note (`storedOverridesEnv`), `OptionsScreen` ÔÜá row, `orchestrator/index.ts` 401 hint, `api-key.test.ts`.
