## Context

Hoje o orquestrador comunica estado via callback direto pra UI. Ex.: ele
instancia um único `onUpdate` callback e envia o `OrchestratorState`
inteiro a cada mudança. Isso é OK para 1 consumer (UI), mas:

- **F1 (janitor)** precisa rodar gates ao final de cada agente.
- **F4 (HMAC audit)** precisa observar *todos* os eventos para chain.
- **F12 (lifecycle hooks)** é literalmente "wrapper exposto desse bus".
- **F14 (Prometheus)** consome bus para métricas.
- **F19 (chaos)** injeta falhas em pontos do bus.

Sem bus, vamos acoplar 5 features 1:1 ao orquestrador → spaghetti.

**Solução:** `EventBus` simples, typesafe, com handlers síncronos
(`pre_*`) e async fire-and-forget (`post_*`). UI vira primeiro subscriber,
não consumer privilegiado.

### Code sketch (`src/orchestrator/event-bus.ts`)

```typescript
import type { AgentStatus, OrchestratorState, IntegrationConflict } from '../lib/types.js';

// --- Event taxonomy ---

export type OrchEvent =
  | { type: 'run_started';       runId: string; baseBranch: string; baseCommit: string; planHash: string; ts: number }
  | { type: 'stage_started';     runId: string; stageIndex: number; stageName: string; ts: number }
  | { type: 'agent_spawned';     runId: string; agentId: number; modelId: string; files: string[]; worktreePath: string; ts: number }
  | { type: 'agent_phase_changed'; runId: string; agentId: number; phase: AgentStatus['phase']; ts: number }
  | { type: 'agent_log';         runId: string; agentId: number; level: 'info' | 'warn' | 'error' | 'debug'; message: string; ts: number }
  | { type: 'agent_finished';    runId: string; agentId: number; success: boolean; costUsd: number; tokensIn: number; tokensOut: number; commitSha?: string; errorKind?: 'timeout' | 'failed'; ts: number }
  | { type: 'stage_completed';   runId: string; stageIndex: number; ts: number }
  | { type: 'merge_started';     runId: string; ts: number }
  | { type: 'merge_conflict';    runId: string; conflict: IntegrationConflict; ts: number }
  | { type: 'merge_completed';   runId: string; finalCommitSha: string; ts: number }
  | { type: 'gate_started';      runId: string; gateName: string; ts: number }
  | { type: 'gate_finished';     runId: string; gateName: string; passed: boolean; durationMs: number; ts: number }
  | { type: 'run_finished';      runId: string; status: 'done' | 'error'; totalCostUsd: number; ts: number };

export type OrchEventType = OrchEvent['type'];

export type EventHandler<E extends OrchEvent = OrchEvent> = (event: E) => void | Promise<void>;

// --- Phase distinction: pre_* gates the next step; post_* is fire-and-forget ---

const PRE_TYPES = new Set<OrchEventType>([
  'agent_spawned',  // pre_task: hooks can abort spawn
  'merge_started',  // pre_merge
]);

// --- Bus ---

export class EventBus {
  private handlers: Map<OrchEventType | '*', Set<EventHandler>> = new Map();

  /** Subscribe to a specific event type, or '*' for all. Returns an unsubscribe fn. */
  on<T extends OrchEventType>(type: T, handler: EventHandler<Extract<OrchEvent, { type: T }>>): () => void;
  on(type: '*', handler: EventHandler<OrchEvent>): () => void;
  on(type: OrchEventType | '*', handler: EventHandler<OrchEvent>): () => void {
    if (!this.handlers.has(type)) this.handlers.set(type, new Set());
    this.handlers.get(type)!.add(handler as EventHandler);
    return () => this.handlers.get(type)?.delete(handler as EventHandler);
  }

  /**
   * Emit an event. For "pre_*" types, awaits handlers serially; if any throws,
   * the throw propagates (caller can abort the upcoming action). For "post_*",
   * handlers run fire-and-forget; errors are logged, not thrown.
   */
  async emit(event: OrchEvent): Promise<void> {
    const handlers = [
      ...(this.handlers.get(event.type) ?? []),
      ...(this.handlers.get('*') ?? []),
    ];
    if (PRE_TYPES.has(event.type)) {
      // Pre: serial, throw propagates
      for (const h of handlers) {
        await h(event);
      }
    } else {
      // Post: parallel fire-and-forget
      await Promise.allSettled(
        handlers.map(async (h) => {
          try {
            await h(event);
          } catch (err) {
            // eslint-disable-next-line no-console
            console.warn(`[event-bus] handler for ${event.type} threw:`, err);
          }
        }),
      );
    }
  }

  /** For tests: drain queue + reset. */
  clear(): void {
    this.handlers.clear();
  }
}

/** Singleton when caller wants global; orchestrator can also instantiate per-run. */
export function createEventBus(): EventBus {
  return new EventBus();
}
```

### Wire into orchestrator (`src/orchestrator/index.ts`)

The orchestrator should accept an optional `bus` in its options. If
`undefined`, create a new one internally. Emit on every state transition:

```typescript
import { createEventBus, type EventBus } from './event-bus.js';

interface OrchestratorOptions {
  // ... existing fields ...
  eventBus?: EventBus;
  onUpdate?: (state: OrchestratorState) => void; // back-compat
}

export class Orchestrator {
  private readonly bus: EventBus;

  constructor(opts: OrchestratorOptions) {
    this.bus = opts.eventBus ?? createEventBus();
    // Bridge to existing onUpdate callback:
    if (opts.onUpdate) {
      this.bus.on('*', () => opts.onUpdate?.(this.snapshotState()));
    }
  }

  /** Public so UI / tests can subscribe to events without touching internals. */
  events(): EventBus { return this.bus; }

  // ... when an agent is spawned:
  await this.bus.emit({
    type: 'agent_spawned',
    runId,
    agentId,
    modelId,
    files,
    worktreePath,
    ts: Date.now(),
  });

  // ... on phase change:
  await this.bus.emit({
    type: 'agent_phase_changed',
    runId, agentId, phase, ts: Date.now(),
  });

  // ... on completion:
  await this.bus.emit({
    type: 'agent_finished',
    runId, agentId, success, costUsd, tokensIn, tokensOut, commitSha, ts: Date.now(),
  });
}
```

import { useEffect } from 'react';

useEffect(() => {

its view from the event stream. This is also more testable (replay events
in tests).

## Tests

### Unit (`src/orchestrator/event-bus.test.ts`)

```typescript
import { describe, it, expect, vi } from 'vitest';
import { createEventBus, type OrchEvent } from './event-bus.js';

const ev = (type: OrchEvent['type'], extra: any = {}): OrchEvent =>
  ({ type, runId: 'r', ts: 0, ...extra } as OrchEvent);

describe('EventBus', () => {
  it('delivers to type-specific subscriber', async () => {
    const bus = createEventBus();
    const fn = vi.fn();
    bus.on('agent_spawned', fn);
    await bus.emit(ev('agent_spawned', { agentId: 1, modelId: 'x', files: [], worktreePath: '/' }));
    expect(fn).toHaveBeenCalledOnce();
  });

  it('delivers to wildcard subscriber', async () => {
    const bus = createEventBus();
    const fn = vi.fn();
    bus.on('*', fn);
    await bus.emit(ev('run_started', { baseBranch: 'main', baseCommit: 'abc', planHash: 'h' }));
    expect(fn).toHaveBeenCalledOnce();
  });

  it('post-event handler errors do not throw', async () => {
    const bus = createEventBus();
    bus.on('agent_finished', () => { throw new Error('boom'); });
    await expect(bus.emit(ev('agent_finished', {
      agentId: 1, success: true, costUsd: 0, tokensIn: 0, tokensOut: 0,
    }))).resolves.toBeUndefined();
  });

  it('pre-event handler errors propagate (abort semantics)', async () => {
    const bus = createEventBus();
    bus.on('agent_spawned', () => { throw new Error('reject'); });
    await expect(bus.emit(ev('agent_spawned', {
      agentId: 1, modelId: 'x', files: [], worktreePath: '/',
    }))).rejects.toThrow('reject');
  });

  it('unsubscribe via returned fn', async () => {
    const bus = createEventBus();
    const fn = vi.fn();
    const off = bus.on('agent_spawned', fn);
    off();
    await bus.emit(ev('agent_spawned', { agentId: 1, modelId: 'x', files: [], worktreePath: '/' }));
    expect(fn).not.toHaveBeenCalled();
  });
});
```

## Acceptance criteria

- [ ] Event bus suporta type-narrowed handlers via TS generics.
- [ ] `pre_*` events: handler throw aborta o caller (orchestrator pode
      cancelar spawn).
- [ ] `post_*` events: handler errors são logados, nunca matam orquestrador.
- [ ] UI consome via `bus.on('*', ...)` em vez de callback direto.
- [ ] Backwards-compat: `onUpdate` callback continua funcionando (bridge no
      construtor).
- [ ] Smoke: `./scripts/smoke-pipeline.sh` passa sem mudanças no script.
- [ ] Cobertura ≥90% no novo arquivo.

## Out of scope

- ❌ Hooks externos via shell scripts (isso é F12).
- ❌ Persistência de eventos em disco (isso é F4 audit log).
- ❌ Backpressure / queue full handling — eventos são síncronos por
      simplicidade. Se um handler é muito lento, é problema do handler.
- ❌ Cross-process pub/sub (não temos múltiplos processos).

## Risk register

| Risco | Mitigação |
|---|---|
| Ordem de eventos sutil | Documentar invariantes: `agent_finished` depois de `agent_phase_changed → done`. Test que verifica essa ordem. |
| Subscriber lento bloqueia bus em modo `pre_*` | Documentar; manter lista de pre_* mínima (só `agent_spawned` e `merge_started`). |
| Race entre orchestrator emit + UI render | React's `useEffect` cleanup já garante unmount-safety; documentar. |
