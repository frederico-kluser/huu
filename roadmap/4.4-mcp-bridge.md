# 4.4 MCP Bridge

> Phase 4 — Intelligence (Act 3: Resolution)

Bridge between Model Context Protocol servers and agent tools. Lazy-start, token-efficient, on-demand discovery.

## Checklist

- [ ] MCP client programmatic setup (`@modelcontextprotocol/sdk`)
- [ ] Bridge: MCP tools → agent custom tools
- [ ] Lazy-start servers (connect on first use, disconnect after idle)
- [ ] Token-efficient proxy pattern (single proxy tool, on-demand discovery)

## Dependencies

- 0.1 Project Scaffolding (@modelcontextprotocol/sdk installed)
- 1.1 Agent Runtime (tool registration)

## Outputs

- `src/mcp/client.ts` — MCP client lifecycle (connect, discover, call, disconnect)
- `src/mcp/bridge.ts` — MCP tools → agent custom tools mapping
- `src/mcp/proxy.ts` — token-efficient proxy pattern

## Technical Notes

- Transport: stdio for local servers (child process, stdin/stdout)
- Lifecycle: initialize → capabilities exchange → notifications/initialized → operation
- Lazy-start: servers spawned on first tool call, disconnected after 10 min idle
- Proxy pattern: single proxy tool (~200 tokens) wraps all MCP tools, discovers on-demand
- Prevents context window bloat from registering all tools upfront (10K+ tokens)
