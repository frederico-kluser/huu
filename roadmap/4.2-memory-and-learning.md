# 4.2 Memory & Learning

> Phase 4 — Intelligence (Act 3: Resolution)

The system learns and improves with every session. Observations → pattern detection → instincts → knowledge promotion.

## Checklist

- [ ] Observation logging via tool call hooks (100% coverage)
- [ ] Pattern detection (threshold: 20+ observations → Haiku analysis)
- [ ] Instinct generation with confidence scores (0.3-0.85)
- [ ] Instinct decay when contradicted by evidence
- [ ] Instinct promotion to project-level knowledge
- [ ] Session summary generation on completion
- [ ] Context loading: last 7 days of sessions on startup

## Dependencies

- 0.2 SQLite Schema (observations, instincts, sessions tables)
- 2.5 Context-Curator Integration (curation logic)

## Outputs

- `src/memory/observer.ts` — tool call observation logging
- `src/memory/analyzer.ts` — pattern detection with Haiku
- `src/memory/instincts.ts` — instinct lifecycle (create, decay, promote)
- `src/memory/sessions.ts` — session summary and loading

## Technical Notes

- Observations are 100% deterministic (logged on every tool call, not probabilistic)
- Pattern detection uses Haiku (cheapest model) — analysis is a mechanical task
- Instinct confidence: 0.3 (weak signal) to 0.85 (strong pattern)
- Decay: instincts lose confidence when contradicted, are removed below 0.2
- Promotion: high-confidence instincts can be promoted to project-level knowledge (entities table)
