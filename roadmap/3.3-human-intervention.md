# 3.3 Human Intervention

> Phase 3 — TUI (Act 2 → Act 3: Resolution)

The human stays in control. Steer, follow-up, abort, or promote — direct interaction with running agents.

## Checklist

- [ ] `[S]teer` — send redirect message to running agent
- [ ] `[F]ollow-up` — queue instruction for after current turn
- [ ] `[A]bort` — cancel agent, discard worktree, move to Failed
- [ ] `[P]romote` — save learning from Done task to instincts

## Dependencies

- 3.2 Detail View (intervention happens from detail view)
- 1.1 Agent Runtime (abort support via AbortController)
- 0.2 SQLite Schema (messages for steer/follow-up)

## Outputs

- `src/tui/components/InterventionBar.tsx` — action buttons with keyboard shortcuts
- Integration with agent runtime for steer/follow_up/abort semantics
- Instinct promotion flow (Done task → instinct file)

## Technical Notes

- Steer = interrupt agent NOW with new instruction (like agent-stuff's steer)
- Follow-up = queue for AFTER current turn (like agent-stuff's follow_up)
- Abort = AbortController.abort() + worktree cleanup
- Promote = extract learning from completed task, save to instincts table
