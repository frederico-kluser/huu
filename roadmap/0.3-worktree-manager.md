# 0.3 WorktreeManager

> Phase 0 — Foundation (Act 1: Setup)

Wrapper around simple-git `raw()` for worktree lifecycle. Each agent gets an isolated filesystem — this class manages creation, cleanup, and safe concurrent access.

## Checklist

- [ ] `WorktreeManager` class wrapping simple-git `raw()` calls
- [ ] `create(agentId, baseBranch)` — creates branch + worktree
- [ ] `remove(agentId)` — removes worktree + optionally deletes branch
- [ ] `list()` — lists active worktrees with status
- [ ] `getGit(agentId)` — returns isolated SimpleGit instance for a worktree
- [ ] Mutex/semaphore for operations on shared refs
- [ ] Symlink support for `node_modules` (avoid redundant installs)
- [ ] Tests with real git repos (temp directories)

## Dependencies

- 0.1 Project Scaffolding (simple-git installed)

## Outputs

- `src/git/WorktreeManager.ts` — typed class with full worktree lifecycle
- Test suite using real temporary git repositories

## Technical Notes

- Each worktree has its own `index.lock` (no conflict between agents)
- All worktrees share `.git/refs/` (merge on main must be serialized)
- One SimpleGit instance per worktree = natural isolation
- Never run `git gc`/`prune` while worktrees are active
