# 2.5 Context-Curator Integration

> Phase 2 — Orchestration (Act 2: Confrontation)

The "writers' assistant" of the showrunner model. Runs after every agent completes, curating what enters the central scratchpad.

## Checklist

- [ ] Post-activity hook: curator runs after every agent completes
- [ ] Scratchpad update logic: what changed, what to add/remove from knowledge base
- [ ] Strategic compact at beat sheet checkpoints
- [ ] Retrieval just-in-time: load relevant context per agent, not everything

## Dependencies

- 2.3 Remaining Agents (context-curator agent definition)
- 2.1 Beat Sheet Engine (checkpoint definitions for strategic compact)
- 0.2 SQLite Schema (entities, relations, sessions tables)

## Outputs

- `src/orchestrator/curator.ts` — post-activity curation logic
- `src/orchestrator/scratchpad.ts` — scratchpad read/write/compact operations
- Integration with orchestrator loop (auto-trigger after task_done)

## Technical Notes

- Context-curator uses Haiku (cheapest model) — curation is a mechanical task
- Curates, not accumulates: decides what to keep, update, or discard
- Strategic compact aligns with beat sheet checkpoints (between acts)
- Retrieval just-in-time: each agent gets only relevant context, not the full scratchpad
